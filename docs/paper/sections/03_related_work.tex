\section{Related Work}
\label{sec:related}

Table~\ref{tab:tool-comparison} summarises the feature landscape.
Tools fall into three categories: BAM editors (BAMSurgeon, SomatoSim\autocite{kuhnemund2017somatomsim}, stochasticSim\autocite{osullivan2023stochasticsim}), de-novo read generators (ART, NEAT, ReSeq, DWGSIM, NGSNGS\autocite{ngsngs2023}), and cancer-evolution simulators (VarSim\autocite{mu2015varsim}, PSiTE\autocite{psyte2020psite}, MOV\&RSim\autocite{movsim2025}, GENOMICON-Seq\autocite{genomiconseq2025}).

Two gaps stand out.
First, UMI and duplex sequencing tools (fgbio\autocite{fgbio}, UMI-tools\autocite{smith2017umitools}, HUMID) lack genome-wide simulation infrastructure and rely on circular benchmarking.
Second, cfDNA analysis tools (DELFI\autocite{cristiano2019delfi}, ichorCNA\autocite{adalsteinsson2017ichorcna}) have no configurable in-silico cfDNA simulator with variant support.

A further limitation pervades BAM editors: deterministic spike-in.
When BAMSurgeon injects a variant at 10\% VAF into 30x data, it modifies exactly 3 reads.
Real sequencing follows a binomial sampling process.
Deterministic data overestimates caller sensitivity\autocite{osullivan2023stochasticsim}.
