\section{Future Work}
\label{sec:future}

Four directions follow directly from current limitations.

\textbf{Targeted region filtering.}
Accepting a BED file of capture targets would allow panel simulations at realistic depths (500x--1000x) on large references.
Currently the tool simulates every position in the reference equally, which makes high-coverage panel runs on chromosome-scale references impractical on a standard workstation.

\textbf{Long-read sequencing.}
Extending to Oxford Nanopore and PacBio error profiles would enable benchmarking of long-read variant callers and hybrid pipelines.
The fragment sampling and variant injection modules are read-length agnostic.
The main work is in quality score modelling.

\textbf{Enhanced error profiles.}
The current \texttt{learn-profile} captures per-cycle quality distributions but does not reproduce the full correlation structure modelled by ReSeq\autocite{schmeing2021reseq}.
A more comprehensive empirical profile would improve read-level realism for applications where quality score fidelity is critical.

\textbf{Cloud-native deployment.}
A Nextflow wrapper distributing regions across a cluster would extend \varforge{} to population-scale benchmarking without changes to the core tool.
