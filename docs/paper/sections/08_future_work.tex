\section{Future Work}
\label{sec:future}

The most important next step is output correctness validation.

\textbf{VAF accuracy validation.}
A chi-squared or Kolmogorov-Smirnov test comparing observed alt-read counts against the expected Binomial$(D, \text{VAF})$ distribution would directly validate the Bernoulli sampling model.
This is straightforward to automate: generate reads at a known VAF, align with BWA, count alt reads in pileup, and test the resulting distribution.

\textbf{cfDNA fragment distribution validation.}
The cfDNA mode uses a four-component Gaussian mixture.
Extracting insert sizes from the BAM output and comparing to the configured parameters would confirm that the fragment model is applied correctly.

\textbf{Downstream variant caller benchmarking.}
The benchmark infrastructure is complete (\texttt{benchmarking/downstreamvc/}).
Four scenarios are defined: SNV baseline (30x), low-VAF ctDNA (30x, 5\% purity), FFPE artefacts (30x), and a targeted panel at 200x.
Execution requires bwa-mem2, GATK Mutect2, and rtg-tools, which are not yet installed on the benchmark machine.
Results, once obtained, will directly validate whether VarForge output passes through a standard somatic calling pipeline with the expected sensitivity and precision.

\textbf{Compute-bound thread scaling.}
Current thread scaling data covers only the I/O-bound 10\,MB, 30x workload where threading degrades performance.
Testing thread scaling for UMI simulation (where per-pair compute cost is higher) would characterise when Rayon parallelism provides real benefit.

\textbf{Long-read sequencing.}
Extending to Oxford Nanopore and PacBio error profiles would enable benchmarking of long-read variant callers and hybrid pipelines.
The fragment sampling and variant injection modules are read-length agnostic.
The main work is in quality score modelling.

\textbf{Enhanced error profiles.}
The current \texttt{learn-profile} subcommand captures per-cycle quality distributions but does not reproduce the full correlation structure modelled by ReSeq\autocite{schmeing2021reseq}.
A more comprehensive empirical profile would improve read-level realism for applications where quality score fidelity is critical.
