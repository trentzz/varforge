\begin{abstract}
\noindent
Benchmarking cancer genomics pipelines requires synthetic datasets with
precisely known ground truth, yet no single tool integrates de-novo read
generation, somatic variant spike-in, tumour modelling, UMI/duplex
simulation, cfDNA fragmentation, and library artefact injection.
We present \varforge, a pure-Rust, single-binary framework that unifies
all of these capabilities under a composable YAML configuration.
\varforge is the first simulator to provide genome-wide UMI and duplex
sequencing simulation, the first to offer in-silico cfDNA fragmentation
with variant spike-in, and the first to combine stochastic VAF sampling
with clonal-tree tumour models, copy number support, and FFPE/oxoG
artefacts.  A streaming parallel architecture achieves
${\sim}28{,}000$ read pairs/s on compute-bound workloads and
${\sim}13{,}400$ read pairs/s at WGS scale, with memory bounded by
channel depth rather than dataset size.
\varforge is MIT-licensed at
\url{https://github.com/trentzz/varforge}.
\end{abstract}

\noindent\textbf{Keywords:} synthetic sequencing data, cancer genomics,
variant simulation, UMI, duplex sequencing, cell-free DNA, Rust

\section{Introduction}
\label{sec:intro}

Rigorous benchmarking of cancer sequencing pipelines---variant callers,
UMI deduplication tools, copy number estimators, and liquid-biopsy
algorithms---requires datasets with complete ground truth.  Real patient
data cannot satisfy this requirement; synthetic simulation therefore
plays an indispensable role.

Cancer sequencing introduces challenges that generic read simulators
were not designed to address: tumour--normal cell mixtures parametrised
by purity and clonal architecture; allele-frequency heterogeneity from
sub-clonal evolution; copy number alterations affecting both coverage
and effective VAF; a spectrum of variant types (SNV, indel, MNV, SV,
CNV); library artefacts such as FFPE deamination and oxidative damage;
and cfDNA fragmentation profiles for liquid biopsy.

Despite a rich ecosystem of individual tools, no single framework covers
this space.  \bamsurgeon~\citep{ewing2015bamsurgeon} spikes variants
into existing BAMs but uses deterministic VAF assignment.
ART~\citep{huang2012art} and NEAT~\citep{stephens2016neat} generate
reads de novo but lack tumour models and variant injection.
ReSeq~\citep{schmeing2021reseq} achieves high read-level realism but
supports no variant or cancer features.  Critically, no genome-wide UMI
or duplex simulator exists, and no tool combines cfDNA fragmentation
with somatic variant spike-in.

We present \varforge, which addresses these gaps in a single binary.
The contributions are:
\begin{enumerate}[leftmargin=*, label=(\roman*), nosep]
  \item A unified pipeline covering de-novo reads, variant injection,
        tumour modelling, UMI/duplex simulation, cfDNA fragmentation,
        and library artefacts---all via YAML configuration.
  \item The first genome-wide UMI and duplex sequencing simulator.
  \item The first in-silico cfDNA generator with somatic variant support.
  \item Stochastic VAF sampling via per-read Bernoulli trials.
  \item A streaming Rust implementation with bounded memory.
\end{enumerate}

\section{Related Work}
\label{sec:related}

Table~\ref{tab:tool-comparison} summarises the feature landscape.
Tools fall into three categories: BAM editors (\bamsurgeon,
SomatoSim~\citep{kuhnemund2017somatomsim},
stochasticSim~\citep{osullivan2023stochasticsim}), de-novo read
generators (ART, NEAT, ReSeq, DWGSIM, NGSNGS~\citep{ngsngs2023}), and
cancer-evolution simulators
(VarSim~\citep{mu2015varsim}, PSiTE~\citep{psyte2020psite},
MOV\&RSim~\citep{movsim2025}, GENOMICON-Seq~\citep{genomiconseq2025}).

Two gaps are critical.  First, UMI and duplex sequencing tools
(fgbio~\citep{fgbio}, UMI-tools~\citep{smith2017umitools}, HUMID)
lack genome-wide simulation infrastructure and rely on circular
benchmarking.  Second, cfDNA analysis tools
(DELFI~\citep{cristiano2019delfi},
ichorCNA~\citep{adalsteinsson2017ichorcna}) have no configurable
in-silico cfDNA simulator with variant support.

A further limitation pervades BAM editors: deterministic spike-in.
When \bamsurgeon injects a variant at 10\% VAF into 30$\times$ data,
it modifies exactly 3 reads.  Real sequencing follows a binomial
sampling process; deterministic data overestimates caller
sensitivity~\citep{osullivan2023stochasticsim}.

\section{Methods}
\label{sec:methods}

\subsection{Architecture Overview}

Figure~\ref{fig:pipeline} illustrates \varforge's pipeline.  A YAML
configuration drives all parameters.  The reference genome is loaded and
shared via \texttt{Arc}; variants come from VCF or stochastic generation.
The genome is partitioned into regions (one per chromosome by default),
each processed by a Rayon~\citep{rayon} worker thread.  Workers send
completed batches through a bounded \texttt{crossbeam-channel}
(default depth 64) to a dedicated writer thread that produces FASTQ,
BAM, and truth VCF output.

\subsection{Read Generation}

Fragment lengths are sampled from $\mathcal{N}(\mu, \sigma^2)$ for
standard libraries or from a cfDNA mixture model
(Section~\ref{sec:cfdna}).  Start positions are drawn uniformly within
each region.  Two quality models are supported: a parametric model with
per-cycle decay ($\bar{Q}(c) = Q_0 - \lambda c$), and an empirical
model learned from a real BAM via the \texttt{learn-profile} subcommand.

\subsection{Stochastic VAF Sampling}

For each read overlapping a variant site, \varforge independently draws:
\begin{equation}
  X_i \sim \text{Bernoulli}(\text{VAF}_\text{eff})
  \label{eq:bernoulli}
\end{equation}
so the alt-read count follows
$A \sim \text{Binomial}(D, \text{VAF}_\text{eff})$, matching the
natural sampling variance of real sequencing.  This contrasts with
\bamsurgeon's deterministic $A = \lfloor D \cdot \text{VAF} \rfloor$.

\subsection{Variant Spike-In}

\textbf{Small variants.}  SNVs replace reference bases; indels modify
sequence and CIGAR with local re-alignment; MNVs are applied atomically
to preserve cis-phasing.

\textbf{Structural variants.}  Deletions, insertions, inversions, tandem
duplications, and translocations are realised via split reads and
discordant pairs with correct CIGAR strings, SA tags, and mate
coordinates.

\textbf{Copy number variants.}  Read depth is scaled by
$\text{depth\_mult} = (p \cdot \text{CN}_\text{tumor} + (1-p) \cdot 2) / 2$,
and effective VAF is adjusted per the full cancer genomics equation.

\subsection{Tumour Modelling}

Heterogeneity is represented as a directed tree of clones, each with a
cancer cell fraction (CCF).  The effective VAF integrates purity,
ploidy, CCF, and copy number:
\begin{equation}
  \text{VAF}_\text{eff} =
    \frac{\text{CCF} \times m \times p}
         {p \cdot \text{CN}_\text{tumor} + (1-p) \cdot \text{CN}_\text{normal}}
  \label{eq:vaf}
\end{equation}

\subsection{UMI and Duplex Simulation}
\label{sec:umi}

Each molecule receives a UMI barcode of configurable length.  Family
sizes follow a log-normal distribution reflecting real PCR kinetics.
In duplex mode~\citep{schmitt2012duplex}, each double-stranded molecule
yields two strand families with swapped UMI pairs $(\alpha,\beta)$ and
$(\beta,\alpha)$.  True mutations appear on both strands; artefacts are
strand-specific, faithfully reproducing the error suppression that makes
duplex sequencing valuable.

\subsection{Cell-Free DNA Fragmentation}
\label{sec:cfdna}

The cfDNA model samples fragment lengths from a four-component Gaussian
mixture:
\begin{equation}
  P(L) = \sum_{k=1}^{4} w_k \cdot \mathcal{N}(L;\, \mu_k,\, \sigma_k^2)
  \label{eq:cfdna}
\end{equation}
with sub-nucleosomal (${\sim}143$\,bp), mono-nucleosomal
(${\sim}167$\,bp), di-nucleosomal (${\sim}334$\,bp), and
tri-nucleosomal (${\sim}500$\,bp) components.  Below 167\,bp, a 10\,bp
sinusoidal periodicity matching the DNA helical repeat is
applied~\citep{snyder2016cfdna_nucleosome}.  Tumour-derived fragments
are shifted shorter (${\sim}143$--147\,bp
peak)~\citep{lo2010cfdna}, enabling fragment-based tumour fraction
estimation by tools such as DELFI.

\subsection{Library Artefacts}

\textbf{FFPE deamination.}  C$>$T transitions at a configurable rate,
preferentially at fragment ends.
\textbf{Oxidative damage.}  G$>$T transversions with strand-specific
bias.
\textbf{PCR errors.}  Per-cycle substitution errors propagated through
family genealogies.
\textbf{GC bias.}  Fragment-level coverage modulation with empirical
U-shaped weighting.

\subsection{BAM Editing Mode}

The \texttt{varforge edit} subcommand spikes variants into existing BAM
files with stochastic VAF, UMI-family-aware read selection, and direct
split-read construction (avoiding \bamsurgeon's local assembly step).

\subsection{Implementation}

\varforge is implemented in Rust with a pure-Rust dependency stack
(noodles~\citep{noodles} for BAM/VCF/FASTA, \texttt{seq\_io} for FASTQ,
\texttt{flate2} for compression).  No C libraries are required,
enabling single-binary distribution.  Deterministic seeding uses
$\text{seed}_r = H(\text{seed}_\text{global}, r)$ per region, ensuring
reproducibility independent of thread scheduling.

\section{Results}
\label{sec:results}

\subsection{Experimental Setup}

All benchmarks ran on an AMD Ryzen~5 7640HS (6 cores / 12 threads),
12\,GiB DDR5, NVMe SSD, Debian~13, \texttt{rustc}~1.94.0 release mode.
Synthetic references of 1\,MB (6 chromosomes) and 10\,MB (8
chromosomes) were generated with ${\sim}40$\% GC content.  Each
configuration was run three times with different seeds; mean values
are reported.  Benchmarks collected March~2026.

\subsection{Coverage Scaling}

Figure~\ref{fig:cov-scaling} shows wall time and peak memory scaling
with coverage depth on the 1\,MB reference with 100 random mutations.
Both metrics scale linearly ($R^2 > 0.999$).  Throughput stabilises at
${\sim}28{,}000$ read pairs/s above $10\times$; the lower rate at
$1\times$ reflects start-up overhead.  Memory consumption is
${\sim}1.7$\,KB per read pair, consistent across all depths
(Figure~\ref{fig:mem-scaling}).

\subsection{Feature Overhead}

Figure~\ref{fig:features} shows the incremental cost of each feature on
the 1\,MB reference at $30\times$ (${\sim}200$K read pairs).  Variant
spike-in ($+3\%$) and GC bias ($+2\%$) are within measurement noise.
FFPE/oxoG artefacts add $16\%$ via a per-base damage pass.  UMI
simulation is the most expensive feature at $+64\%$ due to PCR family
genealogy tracking.  The combined overhead of $125\%$ is slightly
super-additive because UMI families must carry variant and artefact
state through each PCR cycle.

\subsection{Memory Efficiency}

The streaming architecture (Section~\ref{sec:methods}) bounds peak
memory to $\text{buffer\_depth} \times \text{batch\_size}$ regardless of
total dataset size.  For a 1\,Mbp region at $30\times$, each batch
contains ${\sim}100$K pairs at ${\sim}1.7$\,KB each (${\sim}170$\,MB).
With the default 64-slot channel, the theoretical ceiling is
${\sim}10.5$\,GiB.  A full human genome $30\times$ WGS run
(${\sim}600$M pairs) is therefore feasible on a standard workstation
without swap pressure.

\subsection{Comparison with Existing Tools}

Direct throughput comparison is limited by feature asymmetry: no other
tool provides the same capabilities.  \bamsurgeon injects variants at
${\sim}1$ variant per 4--5\,s (${\sim}40$\,min for 500 variants);
\varforge applies 500 variants in $<1$\,s within a 75\,s total run.
ART is single-threaded and lacks variant support.  NEAT is
Python-based with limited WGS-scale throughput.  \varforge is the only
tool offering all capabilities in Table~\ref{tab:tool-comparison}
simultaneously.

\section{Discussion}
\label{sec:discussion}

\varforge addresses three critical gaps in the benchmarking ecosystem.

\textbf{UMI/duplex simulation.}  The rapid clinical adoption of
error-corrected sequencing has outpaced benchmarking infrastructure.
\varforge enables the first systematic evaluation of UMI grouping,
consensus calling, and duplex variant detection tools across the full
parameter space of family size, UMI error rate, and tumour fraction.

\textbf{cfDNA simulation.}  Liquid biopsy tools such as DELFI and
ichorCNA can now be benchmarked against configurable in-silico datasets
spanning tumour fractions from 50\% to $<0.01\%$---a regime where
wet-lab reference materials are prohibitively expensive.

\textbf{Stochastic VAF.}  By replacing deterministic spike-in with
Bernoulli sampling, \varforge eliminates the systematic optimism that
inflates sensitivity estimates, particularly at low VAF and low
coverage.

\textbf{Performance for CI/CD.}  The streaming Rust architecture enables
integration into automated testing pipelines.  The \texttt{small} preset
completes in ${\sim}30$\,s; panel simulations finish in minutes; full
WGS is feasible overnight.

\section{Future Improvements}
\label{sec:future}

\textbf{Long-read sequencing.}  Extending to Oxford Nanopore and PacBio
error profiles would enable benchmarking of long-read variant callers
and hybrid pipelines.

\textbf{Single-cell modes.}  Single-cell DNA/RNA sequencing introduces
allelic dropout, amplification bias, and ambient RNA---features not yet
modelled.

\textbf{Methylation.}  Bisulfite and nanopore methylation simulation
would support benchmarking of differential methylation callers.

\textbf{Enhanced error profiles.}  The current \texttt{learn-profile}
captures per-cycle quality distributions but does not reproduce the
full correlation structure modelled by
ReSeq~\citep{schmeing2021reseq}.  Future work will develop more
comprehensive empirical profiles.

\textbf{Cloud-native deployment.}  A Nextflow wrapper distributing
regions across a cluster would extend \varforge to population-scale
benchmarking.

\textbf{Clonal evolution dynamics.}  Adding stochastic birth/death
processes between longitudinal timepoints would enable benchmarking
of molecular residual disease monitoring algorithms.

\section{Conclusion}
\label{sec:conclusion}

\varforge is a unified, high-performance framework for synthetic cancer
sequencing data generation.  By combining de-novo reads, variant
injection, tumour modelling, UMI/duplex simulation, cfDNA fragmentation,
and library artefacts in a single Rust binary, it eliminates the
multi-tool pipeline assembly that currently bottlenecks cancer genomics
benchmarking.  The streaming architecture bounds memory independently of
dataset size, making full human genome simulation feasible on standard
workstations.
