# T126: Integration tests for BAM editor (varforge edit)

**Priority**: P1
**Depends on**: —
**Status**: done

## Goal

There are no integration tests for the `varforge edit` subcommand. Add tests
that run the BAM editor and assert that edits are applied correctly to the
output BAM.

## Steps

1. Generate a small synthetic BAM for use as input.
2. Run `varforge edit` with a known edit config.
3. Assert that the output BAM contains the expected edits (e.g. tag changes,
   read modifications) and that unaffected records are unchanged.
4. `cargo fmt -- --check && cargo clippy -- -D warnings && cargo test`

## Success criteria

- [ ] Integration test covers at least one edit operation.
- [ ] Output BAM assertions confirm edits were applied.
- [ ] Unedited records are unchanged.
- [ ] `cargo test` passes cleanly.
