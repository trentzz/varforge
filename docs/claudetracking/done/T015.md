# T015: Implement `variant_reads.tsv` sidecar writer

**Epic**: EPIC-OUTPUT (overallplans/EPIC-OUTPUT.md)
**Priority**: high
**Depends on**: T013
**Status**: done

## Goal

Write a `variant_reads.tsv` file alongside the FASTQ output. Each row records a read name and the variant it carries, enabling fast lookup of which reads support each variant.

## Steps

1. Open `variant_reads.tsv` in the output directory at simulation start.
2. For each read pair with non-empty `variant_tags`, write one row per tag.
3. Columns: `read_name`, `chrom`, `pos`, `vartype`, `vaf`, `clone_id`.
4. Close the file after all read pairs are written.
