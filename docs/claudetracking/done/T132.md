# T132: Add automatic tabix index generation after truth VCF output

**Priority**: P2
**Depends on**: —
**Status**: done

## Goal

VarForge does not generate a tabix index (`.tbi`) alongside the truth VCF.
Most downstream tools require a tabix index for region queries. After writing
the VCF, automatically generate the `.tbi` index. Requires VCF to be sorted
(see T114) and bgzf-compressed.

## Steps

1. Ensure the truth VCF is written in bgzf-compressed format (`.vcf.gz`).
2. After the VCF writer finishes, invoke the noodles (or equivalent) tabix
   indexing API to produce a `.tbi` file.
3. Log or report the index path.
4. Add a test that confirms a `.tbi` file is created after simulation.
5. `cargo fmt -- --check && cargo clippy -- -D warnings && cargo test`

## Success criteria

- [ ] Truth VCF output is bgzf-compressed.
- [ ] A `.tbi` file is created automatically alongside the truth VCF.
- [ ] A test asserts the `.tbi` file exists after simulation.
- [ ] `cargo test` passes cleanly.
