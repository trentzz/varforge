## Goal

Fix `src/cli/edit.rs:169` where `cfg.purity.min(purity_override)` takes the minimum of config
and CLI values. The correct behaviour is: CLI flag wins if supplied, config is the fallback.

## Steps

1. Identify how `purity_override` is obtained (likely from `EditOpts`)
2. Change the merge to: if the CLI flag was explicitly set, use the CLI value; otherwise use
   the config value
3. Add a test that sets config purity to 0.8 and CLI purity to 0.3, confirming the result is
   0.3 (CLI wins), not 0.3 (minimum happens to be the same here — pick values where min gives
   wrong answer, e.g. config=0.3, CLI=0.8, expect 0.8, not 0.3)

## Notes

The current `min()` call silently ignores the CLI flag whenever the config purity is lower.
