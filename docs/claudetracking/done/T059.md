# T059: Unify write_pair and write_pair_with_umi in bam.rs

**Epic**: EPIC-REFACTOR
**Priority**: P2
**Depends on**: none
**Status**: done

## Goal

Consolidate `write_pair` and `write_pair_with_umi` in `src/io/bam.rs` into a shared internal
builder, eliminating the ~90% code duplication. The public API stays the same.

## Steps

1. Extract a private `build_record(pair, mate_is_read1, umi: Option<&str>, ...) -> RecordBuf`
   helper that both public functions call
2. `write_pair` calls `build_record` with `umi: None`
3. `write_pair_with_umi` calls `build_record` with `umi: Some(tag)`
4. `cargo test` — existing BAM tests must all pass
5. `cargo clippy -- -D warnings`
