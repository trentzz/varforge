# T133: Migrate from serde_yml to serde_yaml

**Status**: DONE
**Priority**: P2

## Context

`serde_yml` (the community fork of `serde_yaml`) has itself stalled. The original
`serde_yaml` v0.9 crate is more widely used and its advisory (`RUSTSEC-2024-0341`) is
only a deprecation notice, not a vulnerability. Reverting to `serde_yaml = "0.9"` gives
a more stable, better-supported dependency with identical API surface.

## Changes

1. `Cargo.toml`: replaced `serde_yml = "0.0"` with `serde_yaml = "0.9"`.
2. `src/cli/edit.rs`: replaced all `serde_yml::` calls with `serde_yaml::`.
3. `src/io/config.rs`: replaced all `serde_yml::` calls with `serde_yaml::`.
4. `tests/simulate_integration.rs`: replaced all `serde_yml::` calls with `serde_yaml::`.

- `cargo clippy -- -D warnings`: pass
- `cargo test`: 375 unit tests + 33 integration tests, all pass
