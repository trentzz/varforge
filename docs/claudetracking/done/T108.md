# T108: Add duplex conversion rate to sim_report.json

**Epic**: EPIC-TWIST-DUPLEX
**Priority**: P1
**Depends on**: T106
**Status**: done

## Goal

Report the duplex conversion rate in `sim_report.json`: the fraction of unique molecules
(by UMI pair) for which both the AB and BA strand families were observed. This is the
primary quality metric for a duplex library.

## Steps

1. During duplex family generation, track:
   - `total_molecules`: count of unique UMI pairs generated
   - `duplex_molecules`: UMI pairs where both an AB and a BA family were written to output
2. `duplex_conversion_rate = duplex_molecules / total_molecules`
3. Add `duplex_conversion_rate` to `sim_report.json` at the top level
4. Add a per-chromosome breakdown if feasible
5. Add a unit test: with duplex=true and coverage=100, the reported conversion rate is > 0.8
6. `cargo test`

## Notes

In practice, not every molecule produces both strands due to library preparation losses.
In simulation, both strands are always generated unless explicitly dropped. The conversion
rate should be close to 1.0 in simulation but the metric is still useful to verify the
counting logic is correct before comparing against real Twist data.
