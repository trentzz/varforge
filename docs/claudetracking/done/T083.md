# T083: Validate read_length <= fragment mean

**Epic**: EPIC-CONFIG-VALIDATION
**Priority**: P1
**Depends on**: none
**Status**: done

## Goal

Add a validate() check that read_length ≤ fragment.mean. If read_length > fragment.mean,
most fragments will be shorter than one read, producing invalid or degenerate output. Fail
fast with a clear error message.

## Steps

1. In `src/io/config.rs` validate(), add:
   `ensure!(cfg.read_length <= cfg.fragment.mean as usize, "read_length ({}) must be <= fragment.mean ({})", ...)`
2. Add a unit test: a config with read_length=200, fragment.mean=150 fails validation with
   the expected error text
3. `cargo test`
