# T036: Add `haplotype: Option<u8>` to `Variant`; implement haplotype-aware spike-in

**Epic**: EPIC-ADVANCED (overallplans/EPIC-ADVANCED.md)
**Priority**: medium
**Depends on**: none
**Status**: done

## Goal

Each `Variant` can be assigned to haplotype 0 or 1. During spike-in, a variant is only applied if the fragment was sampled from the matching haplotype.

## Steps

1. Add `haplotype: Option<u8>` to the `Variant` struct.
2. During fragment sampling, assign fragments to haplotype 0 or 1 (randomly or deterministically).
3. In spike-in, skip the variant if `variant.haplotype` is set and does not match the fragment's haplotype.
