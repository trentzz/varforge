# T062: Tests for cli/validate.rs, cli/benchmark_suite.rs, cli/learn_profile.rs

**Epic**: EPIC-TESTS
**Priority**: P2
**Depends on**: none
**Status**: done

## Goal

Add unit tests for the three CLI subcommand entry points that currently have none.

## Steps

1. **validate.rs**: test that `run()` returns `Ok` for a valid config and `Err` for an invalid
   one (e.g. missing required fields). Use a temp YAML file.
2. **benchmark_suite.rs**: test that `run()` creates the expected directory structure and
   writes `benchmark_manifest.tsv` with the correct number of rows for a 2-VAF × 2-coverage
   grid. Use a minimal config and a temp output directory.
3. **learn_profile.rs**: test that `run()` with a minimal BAM and `--format json` produces a
   valid JSON file; with `--format tsv` produces a TSV file. Verify the output is non-empty.
4. `cargo test` passes for all three
