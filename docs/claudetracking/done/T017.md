# T017: Apply preset defaults after YAML deserialisation

**Epic**: EPIC-PRESETS (overallplans/EPIC-PRESETS.md)
**Priority**: high
**Depends on**: T016
**Status**: done

## Goal

After loading a YAML config, if a `preset` key is present, fill any unset fields with the preset's defaults. Explicit YAML values always take precedence.

## Steps

1. After `serde` deserialisation, check for a `preset` field in the config.
2. Look up the preset via the table from T016.
3. For each field that is `None` or at its default, apply the preset value.
