# T077: Add explicit cfdna.ctdna_fraction config field

**Epic**: EPIC-CFDNA-SEMANTICS
**Priority**: P1
**Depends on**: T076
**Status**: done

## Goal

Add `cfdna.ctdna_fraction: Option<f64>` to the config so users can specify the circulating
tumour DNA fraction independently of purity. When present, it overrides the purity-derived
value.

## Steps

1. Add `ctdna_fraction: Option<f64>` to a `CfdnaConfig` struct (or inline on `FragmentConfig`)
   in `src/io/config.rs`
2. In the engine, resolve the ctdna_fraction: use the explicit field if set, otherwise fall
   back to purity
3. Validate: if set, must be in [0.0, 1.0]
4. Update the cfdna example YAML to show the new field
5. Add a unit test: explicit ctdna_fraction=0.03 overrides purity=0.5
6. `cargo test`
