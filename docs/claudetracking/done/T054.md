# T054: Implement inject_snv_into_md correctly

**Epic**: EPIC-CORRECTNESS
**Priority**: P1
**Depends on**: none
**Status**: todo

## Goal

Replace the stub implementation of `inject_snv_into_md` in `src/editor/read_modifier.rs` with
a correct one that splits the numeric run at the SNV offset and inserts the reference base.
Output must be a valid MD string parseable by samtools.

## Steps

1. Read the SAM spec MD tag format: `[0-9]+(([A-Z]|\^[A-Z]+)[0-9]+)*`
2. Parse the existing MD string into a sequence of (run_length, mismatch_or_deletion) tokens
3. Walk the token list to find the position of the SNV offset, split the enclosing numeric run
4. Insert the reference base at that position and re-serialise
5. Add unit tests:
   - SNV at position 0 in a fully matching read (MD="150")
   - SNV in the middle of a run (MD="75A74" style — existing mismatch present)
   - SNV at the last base
   - SNV in a read with existing deletions (^-notation)

## Notes

The current implementation just appends the ref base to the string, producing invalid output
like "150T" instead of correctly splitting the run.
