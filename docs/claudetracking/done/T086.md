# T086: Validate copy_number region strings at config load time

**Epic**: EPIC-CONFIG-VALIDATION
**Priority**: P1
**Depends on**: none
**Status**: done

## Goal

Validate that every copy_number region string (e.g. "chr7:55000000-55200000") parses
correctly in validate(). A malformed string currently fails deep in the simulation with an
unhelpful panic or error.

## Steps

1. Write or reuse a `parse_region(s: &str) -> Result<(String, u64, u64)>` parser
2. In validate(), for each entry in `cfg.copy_number`, call parse_region and propagate errors
3. Add a unit test: "chr7:abc-def" fails validation with a clear parse error message
4. `cargo test`
