# T063: Negative and edge-case tests for config variable substitution

**Epic**: EPIC-TESTS
**Priority**: P2
**Depends on**: none
**Status**: done

## Goal

Add tests in `src/io/config.rs` for `substitute_vars()` covering cases that are not currently
exercised: unterminated placeholders, unknown keys, empty values, and adjacent placeholders.

## Steps

1. Unterminated placeholder: `"prefix_${key"` — should return an error or pass through
   unchanged (decide and document which)
2. Unknown key: `"${unknown}"` with no matching `--set` entry — should error with a clear
   message naming the missing key
3. Empty value: `--set key=` substitutes an empty string into the YAML
4. Adjacent placeholders: `"${a}${b}"` with both keys defined — both are replaced correctly
5. Nested-looking placeholder: `"${a${b}}"` — should not recurse (treat `a${b}` as the key
   name, which is likely undefined, or error)
6. `cargo test`
