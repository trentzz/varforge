# T122: Integration tests for multi-sample/longitudinal mode

**Priority**: P1
**Depends on**: —
**Status**: done

## Goal

There are no integration tests covering multi-sample or longitudinal simulation
mode. Add tests that run VarForge with a longitudinal config and assert correct
output for each time point.

## Steps

1. Write an integration test config with two or more longitudinal time points.
2. Run the simulation and assert:
   - Output files are created for each sample/time point.
   - VAF shifts between time points match config.
   - `sim_report.json` captures per-sample metrics.
3. `cargo fmt -- --check && cargo clippy -- -D warnings && cargo test`

## Success criteria

- [ ] Integration test covers at least two longitudinal time points.
- [ ] VAF and output file assertions pass.
- [ ] `cargo test` passes cleanly.
