# T068: Wire driver mutations into cancer presets

**Epic**: EPIC-PRESETS
**Priority**: P2
**Depends on**: none
**Status**: done

## Goal

Make the `DriverMutation` tables in `src/cli/cancer_presets.rs` optionally inject known driver
variants at specific genomic positions when a reference is provided. Add a config flag
`include_driver_mutations: bool` (default false) to each cancer preset.

## Steps

1. Read `src/cli/cancer_presets.rs` driver mutation tables to understand the existing data
   structure (`gene`, `chrom`, `pos`, `ref_allele`, `alt_allele`, `aa_change`)
2. Add `include_driver_mutations: bool` to the preset application logic in `presets.rs`
3. When `include_driver_mutations: true`, convert each `DriverMutation` entry into a
   `Variant` (Snv or Indel as appropriate) at the given position with VAF = clonal VAF
   (typically the dominant clone's VAF)
4. Append these variants to the variant list in `build_somatic_variant_list()`
5. Remove `#[allow(dead_code)]` from the driver tables
6. Add a unit test: apply the LUAD preset with `include_driver_mutations: true`, confirm the
   KRAS G12C variant appears in the variant list at the correct position
7. `cargo test`

## Notes

Driver mutations should be additive to whatever is in the VCF or random_mutations config.
They are not a replacement. Document that coordinates are hg38 in the YAML.
