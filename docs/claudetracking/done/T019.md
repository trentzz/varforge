# T019: Parse optional depth column (col 4) from capture BED

**Epic**: EPIC-COVERAGE (overallplans/EPIC-COVERAGE.md)
**Priority**: medium
**Depends on**: none
**Status**: todo

## Goal

The capture BED parser reads an optional fourth column containing a per-target depth multiplier. When present, the value is stored keyed by target coordinates in `CaptureModel`.

## Steps

1. Update the BED parser to check for a fourth column.
2. If present, parse it as `f64` and store in a `HashMap<target_key, f64>` on `CaptureModel`.
3. If absent, leave the map empty (fall back to existing behaviour).

## Notes

Store as `HashMap<target_key, f64>` in `CaptureModel`. Fall back to log-normal sampling when no stored depth for a target.
