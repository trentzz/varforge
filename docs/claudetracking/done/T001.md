# T001: Fix SV boundary panic: skip too-short post-deletion fragments

**Epic**: EPIC-BUGS (overallplans/EPIC-BUGS.md)
**Priority**: high
**Depends on**: none
**Status**: done

## Goal

The simulator panicked when a deletion SV left a post-deletion fragment shorter than the read length. This task fixes the panic by skipping fragments that are too short to produce a valid read pair.

## Steps

1. Identify the fragment length check in the SV spike-in path.
2. Add a guard: if post-deletion fragment length < minimum read length, discard the fragment and continue.
3. Verify no panic on previously failing inputs.
