# T066: Streaming BAM editor

**Epic**: EPIC-TOOLING
**Priority**: P1
**Depends on**: none
**Status**: done

## Goal

Replace `BamEditor::load_bam` (which reads the entire file into a `Vec<RecordBuf>`) with a
streaming approach that processes one region at a time using the BAM index. This makes the
edit subcommand viable for WGS files.

## Steps

1. Require an index (`.bai`) alongside the input BAM; error with a helpful message if absent
2. For each variant, query the BAI for overlapping reads using noodles `IndexedReader` and the
   variant's region
3. Collect only the reads overlapping the current variant, apply the spike-in, write them
   back (or write them to a temp file for merging)
4. Pass-through reads that do not overlap any variant from the original BAM stream
5. Write the final output BAM in coordinate order; build an index on the output
6. Update `cli/edit.rs` to call the streaming path
7. Integration test: edit a 500 bp region in a small BAM, verify the rest of the BAM is
   unchanged

## Notes

The current in-memory approach works for panel BAMs (a few thousand records) but will OOM on
WGS inputs (hundreds of millions of records). The BAI-based query is the standard noodles
streaming pattern.
