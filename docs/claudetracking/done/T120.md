# T120: Add @PG program record to BAM header

**Priority**: P1
**Depends on**: —
**Status**: done

## Goal

BAM files produced by VarForge do not include an `@PG` (program) header record.
This record is expected by downstream tools (e.g. GATK, samtools) for
provenance tracking. Add an `@PG` record with `ID`, `PN`, `VN`, and `CL` fields.

## Steps

1. Locate where the BAM header is constructed.
2. Add an `@PG` record with:
   - `ID:varforge`
   - `PN:varforge`
   - `VN:<crate version from Cargo.toml>`
   - `CL:<command line string>`
3. Add a test that checks the `@PG` record is present in BAM output.
4. `cargo fmt -- --check && cargo clippy -- -D warnings && cargo test`

## Success criteria

- [ ] `@PG` record with `ID`, `PN`, `VN`, and `CL` is present in BAM headers.
- [ ] A test asserts the `@PG` record is present.
- [ ] `cargo test` passes cleanly.
