# T009: Add `--dry-run` flag to simulate subcommand

**Epic**: EPIC-BATCH (overallplans/EPIC-BATCH.md)
**Priority**: high
**Depends on**: none
**Status**: done

## Goal

A `--dry-run` CLI flag causes the simulate subcommand to skip all file I/O. The engine still computes expected counts but writes nothing to disk.

## Steps

1. Add `--dry-run` boolean flag to the simulate subcommand in the CLI definition.
2. Thread the flag through to the engine.
3. In the engine, gate all file writes on `!dry_run`.

## Notes

CLI flag only. When set, all I/O is skipped.
