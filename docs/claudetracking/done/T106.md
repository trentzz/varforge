# T106: Wire duplex_alt_count (track AB+BA concordance per molecule)

**Epic**: EPIC-TWIST-DUPLEX
**Priority**: P0
**Depends on**: none
**Status**: done

## Goal

Fix the `duplex_alt_count` field in `AppliedVariant` which is hardcoded to 0 with
`// TODO(T004): wire duplex family count here` in `src/core/engine.rs:758`. This field is
the number of molecules where the alt allele appears in **both** the AB and BA strand
families — the key signal for duplex variant calling.

## Steps

1. During duplex family generation in `generate_region()`, when a variant is spiked into an
   AB family read, check whether the corresponding BA family (same molecule, same UMI pair)
   also carries the alt. If both carry it, increment a per-variant `duplex_alt` counter.
2. The simplest approach: track a `HashSet<molecule_id>` of AB-alt molecules and BA-alt
   molecules per variant. At the end, `duplex_alt_count = |AB_alt ∩ BA_alt|`.
3. Wire the count into `AppliedVariant.duplex_alt_count`
4. Remove the `// TODO(T004)` comment
5. Add a unit test: at VAF=0.5 and duplex=true, simulate 200 molecules and check that
   `duplex_alt_count > 0` and `duplex_alt_count <= n_alt_mol`
6. `cargo test`

## Notes

A molecule where only the AB strand carries the alt is a sequencing error or a simplex
artefact, not a real variant in duplex calling. This counter is what fgbio and HUMID use
to distinguish true duplex support from simplex noise.
