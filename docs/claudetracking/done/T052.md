# T052: Fix quality averaging operator precedence

**Epic**: EPIC-CORRECTNESS
**Priority**: P0
**Depends on**: none
**Status**: done

## Goal

Fix the operator precedence bug in two files where `(a as u16 + b as u16) as u8 / 2` casts
to u8 before dividing, corrupting quality values above 127. After the fix, tests confirm
values above 127 average correctly.

## Steps

1. `src/variants/structural.rs:191` — wrap the sum in parens: `((a as u16 + b as u16) / 2) as u8`
2. `src/editor/read_modifier.rs:168` — same fix
3. Add a unit test in each module that averages two values > 127 and checks the result

## Notes

The incorrect form: `(a as u16 + b as u16) as u8 / 2` — `as u8` binds tighter than `/`.
Typical Phred scores (0–41) are not affected, but empirical quality profiles and long-read
platforms can produce values > 41 in the raw u8 encoding.
