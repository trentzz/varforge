# T051: Serialise/deserialise quality profile to `.profile.tsv`

**Epic**: EPIC-TOOLING (overallplans/EPIC-TOOLING.md)
**Priority**: low
**Depends on**: T050
**Status**: done

## Goal

The quality profile learned in T050 is serialised to a `.profile.tsv` file. The `quality.profile_path` config field loads this file at runtime, replacing the built-in quality model.

## Steps

1. Define the `.profile.tsv` format: tab-separated, rows = cycles, columns = quality bins.
2. Write a serialiser that outputs the count matrix from T050.
3. Write a deserialiser that reads the TSV back into the count matrix.
4. In the engine, when `quality.profile_path` is set, load and use the profile instead of the default model.

## Notes

Tab-separated; `quality.profile_path` loads it at runtime.
