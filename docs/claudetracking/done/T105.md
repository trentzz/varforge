# T105: Add dedicated `twist` preset with Twist-specific parameters

**Epic**: EPIC-TWIST-DUPLEX
**Priority**: P1
**Depends on**: none
**Status**: done

## Goal

Add a `twist` preset to `src/cli/presets.rs` that reflects the actual Twist Biosciences
hybrid-capture duplex UMI panel parameters. The existing `umi` preset is generic; this one
is tuned to Twist's published specifications.

## Steps

1. Add `"twist"` to the preset match in `apply_preset()` and to `all_names()`
2. Set these parameters based on Twist specifications:
   - `coverage: 2000.0` (Twist recommends 2000× for ctDNA applications)
   - `read_length: 150`
   - `fragment.mean: 170, fragment.sd: 30` (typical cfDNA fragment after size selection)
   - `umi.length: 9, umi.duplex: true, umi.inline: true` (Twist uses 9 bp inline dual UMIs)
   - `umi.family_size_mean: 3.5, umi.family_size_sd: 1.5` (Twist published family sizes)
   - `capture.coverage_uniformity: 0.15` (Twist panels: CV < 0.2)
   - `capture.off_target_fraction: 0.03` (Twist on-target rate >95%)
   - `mutations.random.vaf_min: 0.0001, vaf_max: 0.1` (ultra-low VAF ctDNA range)
   - `mutations.random.count: 50` (panel-scale)
3. Add the `twist` name to the error message listing valid presets
4. Add a unit test: `apply_preset("twist", config)` sets coverage=2000 and duplex=true
5. `cargo test`
