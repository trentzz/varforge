# T065: Unify edit VCF parser with io/vcf_input.rs

**Epic**: EPIC-TOOLING
**Priority**: P1
**Depends on**: none
**Status**: done

## Goal

Replace the bespoke VCF parser in `src/cli/edit.rs` with a call to `io/vcf_input.rs`
`load_variants_from_vcf()`. The edit path should gain MNV support, bgzip reading, CLONE field
parsing, and REF validation for free.

## Steps

1. Read `src/cli/edit.rs` `load_variants_from_vcf` (lines ~227–303) and
   `src/io/vcf_input.rs` `load_variants_from_vcf` to understand the interface differences
2. Update `cli/edit.rs` to call `io::vcf_input::load_variants_from_vcf` instead
3. Map the returned `Variant` list to the form expected by the editor (check if clone_id,
   haplotype, etc. need defaults)
4. Delete the duplicate parser from `cli/edit.rs`
5. Add a test: load a bgzipped VCF with an MNV through the edit path and confirm it is parsed
6. `cargo test`

## Notes

The old parser classifies equal-length REF/ALT with len>1 as Indel, not Mnv. The unified
parser handles this correctly.
