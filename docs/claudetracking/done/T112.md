# T112: Write Twist duplex benchmark scenario config and success criteria doc

**Epic**: EPIC-TWIST-DUPLEX
**Priority**: P2
**Depends on**: T105, T107, T108
**Status**: done

## Goal

Create a runnable example YAML config for a Twist duplex benchmarking scenario and a
document in `docs/` defining what success looks like. This is the reference configuration
for evaluating fgbio, HUMID, and Mutect2 duplex on VarForge-generated data.

## Steps

1. Create `examples/twist_duplex_benchmark.yaml` using the `twist` preset (T105) with:
   - 50 SNVs at VAF 0.001–0.05
   - 10 indels at VAF 0.005–0.05
   - 5 SVs (2 deletions, 2 duplications, 1 translocation) at VAF 0.01–0.1
   - A representative 100-target capture BED (can be synthetic chr1 regions)
   - `${reference}` and `${output_dir}` placeholders
2. Create `docs/features/twist-duplex-benchmarking.md` with:
   - Overview of the Twist duplex sequencing model
   - The success metrics table from the epic (conversion rate, concordance, VAF accuracy,
     uniformity targets)
   - How to run the scenario
   - How to interpret sim_report.json for each metric
   - Example fgbio/HUMID commands for consensus calling on the output
3. `cargo test` (confirms the YAML parses)
