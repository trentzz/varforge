# T148: Wire UMI sequencing error injection into production path

**Epic**: EPIC-KAM-COMPAT
**Priority**: P1
**Depends on**: none
**Status**: done

## Goal

Add `umi.error_rate: Option<f64>` config field and wire the existing
`inject_umi_errors` function (currently test-only) into the production UMI
generation path. When enabled, each generated UMI has substitution errors
injected at the specified rate before being written to the inline prefix or
read name.

## Success Criteria

- [ ] `inject_umi_errors` is no longer `#[cfg(test)]`-only; it is exported
      from `src/umi/barcode.rs` as a public function.
- [ ] `UmiConfig` has `error_rate: Option<f64>`, default 0.0.
- [ ] Validation rejects `error_rate` outside [0.0, 1.0].
- [ ] In `expand_umi_families`, after generating each UMI, call
      `inject_umi_errors` if `error_rate > 0.0`. Apply independently to each
      copy's UMI (not shared across the family).
- [ ] A unit test generates UMIs with `error_rate = 0.5` and checks that the
      error rate in the output is within [0.4, 0.6].
