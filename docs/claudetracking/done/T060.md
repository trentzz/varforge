# T060: Tests for src/editor/bam_editor.rs

**Epic**: EPIC-TESTS
**Priority**: P1
**Depends on**: none
**Status**: done

## Goal

Add a `#[cfg(test)]` module to `src/editor/bam_editor.rs` covering the core logic:
position index building, overlap detection, and variant spike-in selection.

## Steps

1. Write a helper that builds a minimal in-memory BAM (use noodles RecordBuf directly, no
   file I/O required)
2. Test `build_position_index`: given 3 records at known positions, index contains the right
   offsets
3. Test `find_overlapping_reads`: a read spanning a variant position is returned; a read that
   ends before the position is not
4. Test that a read with a CIGAR containing a deletion is still found when its reference span
   covers the variant position
5. `cargo test` passes

## Notes

The BAM editor has no unit tests at all. Bugs in overlap detection produce incorrect spike-in
(wrong reads modified) with no signal to the user.
