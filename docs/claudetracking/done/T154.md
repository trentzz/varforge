# T154: StrandBiasModel and CorrelatedErrorModel

**Epic**: EPIC-ERROR-MODEL
**Priority**: P2
**Depends on**: T151
**Status**: done

## Goal

Model two instrument-specific noise patterns:
1. R2 has higher error rate than R1 (phasing accumulation over more synthesis
   cycles).
2. Cluster phasing failures cause bursts of correlated errors: 2–5 adjacent
   bases all miscalled to the same wrong base.

## Config API

```yaml
quality:
  sequencing_errors:
    strand_bias:
      r2_error_multiplier: 1.3    # R2 error rate = R1 × this
      r2_quality_offset: -2       # Shift R2 quality scores down by N Phred
    correlated_errors:
      burst_rate: 0.001           # Per-base probability of starting a burst
      burst_length_mean: 3.0      # Mean burst length (geometric distribution)
```

## Success Criteria

- [x] `StrandBiasModel` and `CorrelatedErrorModel` structs in `seq_errors.rs`.
- [x] Strand bias applies `r2_quality_offset` to R2 quality array before error
      injection, and `r2_error_multiplier` to per-base error probabilities.
- [x] Bursts: at each position roll against `burst_rate`; if triggered, draw
      burst length from Geometric(1/burst_length_mean); force all burst
      positions to the same randomly chosen wrong base; set quality to Q12.
- [x] Unit test: R2 mismatch rate is 1.2–1.4× R1 rate with default multiplier.
- [x] Unit test: burst length distribution matches Geometric expectation.
- [x] Unit test: all bases in a burst have the same substitution.
- [x] `cargo test` passes.
