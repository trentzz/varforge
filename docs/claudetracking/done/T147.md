# T147: Add configurable duplex conversion rate

**Epic**: EPIC-KAM-COMPAT
**Priority**: P1
**Depends on**: none
**Status**: done

## Goal

Add `umi.duplex_conversion_rate: f64` config field (range 0.0–1.0). For each
molecule in duplex mode, sample Bernoulli(duplex_conversion_rate) to decide
whether to emit the BA strand family. This makes the simulated duplex
conversion rate realistic (~90% for Twist) instead of always 100%.

## Success Criteria

- [ ] `UmiConfig` has `duplex_conversion_rate: Option<f64>`, default 1.0.
- [ ] Validation rejects values outside [0.0, 1.0].
- [ ] In `expand_umi_families`, when `duplex_conversion_rate < 1.0`, a
      Bernoulli draw determines whether the BA strand is emitted.
- [ ] `duplex_molecules_with_both_strands` is incremented only when both
      strands are emitted.
- [ ] `sim_report.json` `duplex_conversion_rate` reflects the actual observed
      rate, not the config value.
- [ ] A unit test sets conversion rate to 0.5 over 10000 molecules and
      checks the observed rate is within [0.45, 0.55].
