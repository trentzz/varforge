# T091: Implement position-dependent PacBio HiFi quality curve

**Epic**: EPIC-QUALITY-MODELS
**Priority**: P2
**Depends on**: none
**Status**: done

## Goal

Replace the uniform random quality in `src/core/quality.rs` for PacBio HiFi with a
position-dependent curve. Real HiFi reads show Q30-Q40 in the interior and somewhat lower
quality near both ends due to subread stitching.

## Steps

1. Define a simple parametric curve: quality peaks at Q35 in the middle and drops to Q25
   at both ends over ~50 bp, following a smooth cosine or linear interpolation
2. Implement `sample_pacbio_hifi_qualities_positional(length: usize, rng: &mut R) -> Vec<u8>`
3. Replace the current uniform model in the PacBio branch of the engine
4. Add a unit test: mean quality over 10,000 samples at position 0 is < mean at position
   length/2
5. Cite: PacBio SMRT sequencing quality specification (Q20 minimum, median Q35)
6. `cargo test`
