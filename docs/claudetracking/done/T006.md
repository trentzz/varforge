# T006: Add `vafs: Option<Vec<f64>>` field to `Config`

**Epic**: EPIC-BATCH (overallplans/EPIC-BATCH.md)
**Priority**: high
**Depends on**: none
**Status**: todo

## Goal

The `Config` struct gains a `vafs` field. When set, the list of VAFs drives batch mode and the per-variant `vaf_min`/`vaf_max` random range is ignored.

## Steps

1. Add `vafs: Option<Vec<f64>>` to the `Config` struct in `io/config.rs`.
2. Add the corresponding YAML key `vafs`.
3. Update config validation to reject negative or out-of-range VAF values.
4. Update docs/examples to show the field.

## Notes

YAML key is `vafs`. When set, `mutations.random.vaf_min` and `vaf_max` are ignored.
