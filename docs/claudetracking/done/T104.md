# T104: Wire PcrFamilySizeSampler into UMI family generation

**Epic**: EPIC-DEAD-CODE
**Priority**: P2
**Depends on**: none
**Status**: todo

## Goal

Replace the flat or ad-hoc UMI family size logic with `PcrFamilySizeSampler` so that the
number of reads per UMI family follows a realistic log-normal distribution. This makes
VarForge output more useful for benchmarking fgbio, HUMID, and UMI-tools, which all assume
log-normal family size distributions.

## Steps

1. Read `src/umi/families.rs` to understand how family sizes are currently determined
2. Add `family_size_mean: f64` and `family_size_sd: f64` to `UmiConfig` in `src/io/config.rs`
   (sensible defaults: mean=3.0, sd=1.5 based on typical PCR duplication)
3. Construct a `PcrFamilySizeSampler` from those values in the simulation setup
4. Use `sampler.sample(rng)` to draw the number of reads per UMI family instead of the
   current logic
5. Remove the `#[allow(dead_code)]` annotations from `PcrFamilySizeSampler` and its impl
6. Add a unit test: with mean=3.0 and sd=1.5, the mean family size across 1000 families is
   within 1.0 of 3.0, and no family has size 0
7. Update the amplicon preset to use a higher mean family size (e.g. mean=8.0, sd=3.0) to
   reflect amplicon PCR amplification
8. `cargo test`

## Notes

The sampler is already implemented and tested in `src/core/fragment.rs`. This task is purely
about wiring it in and exposing the parameters through config.
