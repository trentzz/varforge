# T096: Extend validate subcommand to check file existence

**Epic**: EPIC-CLI-ERGONOMICS
**Priority**: P2
**Depends on**: T083, T084, T085, T086, T088
**Status**: done

## Goal

After the T083-T088 validation checks are added to `config.rs`, extend the `validate`
subcommand to also check: reference FASTA readability, regions BED parse, and VCF header
parse. These are "pre-flight" checks that catch problems before a long run starts.

## Steps

1. In `src/cli/validate.rs`, after `config::validate(&cfg)`, add:
   - Try to open and read the first record from the reference FASTA
   - If regions_bed is set, parse all lines and report the count and any parse errors
   - If mutations.vcf_path is set, parse the VCF header and report contig count
2. Print a structured summary: "Config OK. Reference: readable. BED: 125 targets. VCF: 42
   variants."
3. `cargo test`
