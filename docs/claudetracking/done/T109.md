# T109: Validate SV reads appear in both AB and BA strand families at breakpoints

**Epic**: EPIC-TWIST-DUPLEX
**Priority**: P1
**Depends on**: T106
**Status**: todo

## Goal

For each simulated SV (deletion, duplication, translocation), verify that reads spanning
the breakpoint appear in both AB and BA strand families of the same molecule. If a BA family
molecule carries a deletion junction in R1, the corresponding AB family R2 should also span
the breakpoint.

## Steps

1. When applying an SV to a read pair in the engine, if duplex mode is active, apply the
   same SV to the BA strand pair (currently BA is generated by revcomp of the AB reads —
   confirm the revcomp preserves the variant sequence correctly)
2. Track `sv_duplex_alt_count` per SV variant (molecules with both strands at breakpoint)
3. Add `sv_strand_concordance` to sim_report.json
4. Add a unit test: apply a 500 bp deletion in duplex mode; confirm at least one AB read
   and one BA read from the same molecule span the breakpoint
5. `cargo test`

## Notes

SVs that only appear in one strand will be filtered by duplex callers as artefacts. This
is the most common cause of false negatives when benchmarking SV callers on duplex data.
