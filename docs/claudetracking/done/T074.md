# T074: Guard template_length i32 overflow for long-read fragments

**Epic**: EPIC-BAM-FIDELITY
**Priority**: P2
**Depends on**: none
**Status**: todo

## Goal

`src/io/bam.rs:140` casts `pair.fragment_length as i32`. PacBio/ONT reads can be tens of
kilobases; while true i32 overflow (> 2 GiB) is unlikely in practice, the cast is unchecked.
Add a saturating cast and a clippy-visible comment.

## Steps

1. Replace `pair.fragment_length as i32` with
   `i32::try_from(pair.fragment_length).unwrap_or(i32::MAX)`
2. Do the same for PNEXT template length fields
3. Add a unit test with fragment_length = u64::MAX, confirm no panic
4. `cargo test`
