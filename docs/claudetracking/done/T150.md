# T150: Integration tests for inline UMI, duplex conversion rate, and UMI errors

**Epic**: EPIC-KAM-COMPAT
**Priority**: P1
**Depends on**: T146, T147, T148, T149
**Status**: done

## Goal

Add integration tests that verify the three new features end-to-end using a
small synthetic simulation run.

## Success Criteria

- [ ] Integration test `inline_umi_r1_r2_prefix`: runs a small simulation with
      `umi.inline: true`, reads back the FASTQ, and asserts every R1 and R2
      record starts with the expected UMI+spacer prefix.
- [ ] Integration test `duplex_conversion_rate_respected`: runs 1000 molecules
      with `duplex_conversion_rate: 0.7` and checks sim_report.json shows a
      rate within [0.65, 0.75].
- [ ] Integration test `umi_error_rate_produces_mutations`: runs with
      `error_rate: 0.5` and checks UMI bases in reads differ from the clean
      UMI at roughly 50%.
- [ ] All tests pass under `cargo test`.
