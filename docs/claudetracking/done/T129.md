# T129: Integration tests for SV simulation and truth VCF output

**Priority**: P1
**Depends on**: —
**Status**: done

## Goal

There are no integration tests covering structural variant (SV) simulation and
the corresponding truth VCF output. Add tests that run VarForge with SV config
and assert that SVs are represented in both reads and the truth VCF.

## Steps

1. Write an integration test config with one or more SVs (deletion, duplication,
   translocation).
2. Run the simulation and assert:
   - Split reads or discordant pairs are present in the BAM output.
   - The truth VCF contains the expected SV records with correct INFO fields.
   - VCF records are sorted (see T114).
3. `cargo fmt -- --check && cargo clippy -- -D warnings && cargo test`

## Success criteria

- [ ] Integration test covers at least deletion and duplication SV types.
- [ ] Truth VCF assertions confirm SV records are present and correct.
- [ ] `cargo test` passes cleanly.
