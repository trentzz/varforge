# T067: Integrate ClonalTree into simulation engine

**Epic**: EPIC-ADVANCED
**Priority**: P2
**Depends on**: none
**Status**: done

## Goal

Wire `src/tumour/clonal_tree.rs` into the simulation engine so that clone hierarchies are
validated and mutation inheritance is enforced. Currently `ClonalTree` is dead code.

## Steps

1. Read `src/tumour/clonal_tree.rs` in full to understand the existing API
2. In `cli/simulate.rs` (or `core/engine.rs`), after building the clone list from config,
   construct a `ClonalTree` from the clone definitions
3. Validate that VAFs are consistent with the tree: a child clone's VAF must be ≤ its
   parent's VAF (or ≤ parent × purity, depending on the model)
4. Use the tree to propagate mutations: variants assigned to a parent clone are also present
   in all descendant clones at the appropriate VAF
5. Remove the `#[allow(dead_code)]` annotations from `clonal_tree.rs`
6. Add a unit test: three-clone tree (root → child → grandchild), verify variant in root
   appears in all three, variant in child appears in child and grandchild but not in an
   unrelated sibling
7. `cargo test`

## Notes

The current flat clone list in config has no inheritance semantics. Two clones with a
parent-child relationship share no mutations unless the user duplicates them manually.
