# T092: Implement homopolymer-aware Nanopore R10 quality degradation

**Epic**: EPIC-QUALITY-MODELS
**Priority**: P2
**Depends on**: none
**Status**: todo

## Goal

Nanopore systematic errors concentrate in homopolymer runs (polyA, polyC, etc.). The current
uniform quality model does not capture this. Reduce quality by a configurable amount in
homopolymer stretches of length ≥ 4.

## Steps

1. Add a helper `is_homopolymer_run(seq: &[u8], pos: usize, min_len: usize) -> bool`
2. In `sample_nanopore_r10_qualities(length, seq, rng)`, scan for homopolymer runs and
   apply a quality penalty (e.g. reduce by 5-10 Phred units, floor at Q10)
3. Update the engine to pass the read sequence when sampling Nanopore qualities
4. Add a unit test: a 10-bp poly-A run at position 50 in a 200-bp read produces lower
   mean quality at positions 50-59 than at positions 100-109
5. `cargo test`
