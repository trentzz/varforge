# T134: Reject inline UMI config with a clear error

**Epic**: EPIC-RELEASE-GATE (overallplans/EPIC-RELEASE-GATE.md)
**Priority**: high
**Depends on**: none
**Status**: done

## Goal

When a user sets `umi.inline: true` in their YAML config, VarForge currently
accepts the value silently and does nothing with it. The inline UMI feature is
not implemented (deferred per vision docs). The config validator must reject
this setting with an actionable error message.

## Success Criteria

- [ ] `validate()` in `src/io/config.rs` checks `umi.inline` and returns an error if `true`.
- [ ] Error message states that inline UMI mode is not supported in v1 and suggests using the default tag-based UMI mode.
- [ ] A unit test verifies the validation rejects `inline: true`.
- [ ] `/update` has been run after changes.

## Steps

1. Add a check in `validate()` for `config.umi.inline == true`.
2. Return `anyhow::bail!("inline UMI mode is not supported in v1. Remove `umi.inline: true` from your config or set it to false.")`.
3. Add a unit test.

## Notes

The `umi.inline` field should remain in the struct for forward compatibility.
Only the validation needs to guard against it.
