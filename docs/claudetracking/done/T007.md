# T007: Implement per-VAF config forking in simulate command

**Epic**: EPIC-BATCH (overallplans/EPIC-BATCH.md)
**Priority**: high
**Depends on**: T006
**Status**: todo

## Goal

When `vafs` is set in the config, the simulate command clones the config once per VAF, overrides the VAF, sets the output subdirectory to `{output}/{vaf_pct}pct/`, and runs each clone in sequence.

## Steps

1. In the simulate command handler, check for `config.vafs`.
2. For each VAF value, clone the config and set the single VAF and the output path.
3. Run the simulation for each cloned config.
4. Report progress per VAF run.

## Notes

Output subdir format: `{output}/{vaf_pct}pct/` where `vaf_pct` is the VAF expressed as a percentage (e.g. `1pct`, `5pct`).
