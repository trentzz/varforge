# T013: Add `variant_tags: Vec<VariantTag>` to `ReadPair`; populate during spike-in

**Epic**: EPIC-OUTPUT (overallplans/EPIC-OUTPUT.md)
**Priority**: high
**Depends on**: none
**Status**: todo

## Goal

Each `ReadPair` carries a list of `VariantTag` structs populated during spike-in. This enables downstream writers to annotate read names and produce the sidecar TSV.

## Steps

1. Define `VariantTag` struct with fields: `chrom`, `pos`, `vartype`, `vaf`, `clone_id`.
2. Add `variant_tags: Vec<VariantTag>` to `ReadPair`.
3. During spike-in, push a `VariantTag` for each variant the read carries.
