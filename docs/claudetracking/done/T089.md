# T089: Generate MD tag for simulated BAM reads

**Epic**: EPIC-MD-TAG
**Priority**: P1
**Depends on**: none
**Status**: todo

## Goal

Set the MD tag on every BAM record produced by the simulate path. VarDict requires MD;
Picard ValidateSamFile flags its absence. For reads with no variants, MD is just the read
length as a number (e.g. "150"). For reads with a spiked SNV at offset 50, MD is "50A99".

## Steps

1. In `src/io/bam.rs`, add a `build_md_string(seq: &[u8], ref_seq: &[u8]) -> String` helper
   that compares read bases to the reference and constructs the MD string
2. In `write_pair` (and UMI/single-read variants), pass the reference sequence slice for
   each read and call `build_md_string`
3. Set `MD:Z:<string>` in the record data fields
4. Unit tests:
   - All-match read (150bp): MD="150"
   - One SNV at offset 50 (ref=A, read=T): MD="50A99"
   - Two SNVs at offsets 10 and 20: MD="10A9C119" (example)
5. `cargo test`

## Notes

The reference sequence slice for each read is already available in the engine (fetched from
the reference genome). It needs to be passed through to the BAM writer, which may require
a signature change on write_pair.
