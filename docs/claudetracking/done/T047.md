# T047: Implement `${variable}` substitution in Config loading + `--set key=value` CLI flag

**Epic**: EPIC-TOOLING (overallplans/EPIC-TOOLING.md)
**Priority**: low
**Depends on**: none
**Status**: done

## Goal

YAML config files support `${variable}` placeholders. The `--set key=value` CLI flag injects values for these placeholders before deserialisation, enabling config reuse across parameter sweeps.

## Steps

1. Pre-process the raw YAML string before passing it to serde: replace `${key}` with the corresponding `--set` value.
2. Add `--set key=value` (repeatable) to the simulate subcommand CLI.
3. Collect `--set` pairs into a `HashMap<String, String>` and pass to the pre-processor.
4. Error clearly if a placeholder has no corresponding `--set` value.
