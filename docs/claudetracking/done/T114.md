# T114: Fix truth VCF sorting for tabix compatibility

**Priority**: P0
**Depends on**: —
**Status**: done

## Goal

Truth VCF records are emitted in an unsorted order. Tabix requires records to
be sorted by position within each contig. Fix the VCF writer to emit records in
sorted order.

## Steps

1. Identify where truth VCF records are collected and written.
2. Sort records by (contig, position) before writing.
3. Confirm the output passes `bcftools sort --check` or equivalent validation.
4. Add a test that asserts output records are in sorted order.
5. `cargo fmt -- --check && cargo clippy -- -D warnings && cargo test`

## Success criteria

- [ ] Truth VCF output is sorted by (contig, position).
- [ ] A test asserts sorted order in the output.
- [ ] `cargo test` passes cleanly.
