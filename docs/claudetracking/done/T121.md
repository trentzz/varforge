# T121: Clean up dead_code annotations

**Priority**: P1
**Depends on**: —
**Status**: done

## Goal

The codebase has a number of `#[allow(dead_code)]` annotations suppressing
warnings for unused items. Each annotation should be resolved: either remove
the unused code, or add a comment explaining why the item must be retained
(e.g. it is part of a public API or reserved for a planned feature).

## Steps

1. Search for all `#[allow(dead_code)]` annotations in `src/`.
2. For each annotation, determine if the item is genuinely unused:
   - If unused and not needed: delete it.
   - If retained intentionally: add a comment explaining why.
3. Remove annotations that are no longer needed after deletions.
4. `cargo fmt -- --check && cargo clippy -- -D warnings && cargo test`

## Success criteria

- [ ] No unexplained `#[allow(dead_code)]` annotations remain.
- [ ] All retained annotations have an explanatory comment.
- [ ] `cargo clippy -- -D warnings` passes cleanly.
- [ ] `cargo test` passes cleanly.
