# T084: Validate capture BED path exists when capture mode is set

**Epic**: EPIC-CONFIG-VALIDATION
**Priority**: P1
**Depends on**: none
**Status**: done

## Goal

If `capture.bed_path` is set, validate that the file exists at config load time. Currently
a missing BED file fails at simulation runtime with a generic I/O error.

## Steps

1. In validate(), if `cfg.capture.is_some()` and `capture.bed_path.is_some()`, check the
   path exists with `std::path::Path::exists()`
2. Return an error with the full path in the message
3. Add a unit test: capture config pointing to a nonexistent path fails validation
4. `cargo test`
