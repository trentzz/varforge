# T153: KmerErrorModel — context-dependent error multipliers

**Epic**: EPIC-ERROR-MODEL
**Priority**: P1
**Depends on**: T151
**Status**: done

## Goal

Certain sequence contexts are systematically error-prone. Implement a k-mer
hash table of per-context error multipliers so homopolymers, poly-G, and GGC
motifs elevate error rates realistically.

## Config API

```yaml
quality:
  sequencing_errors:
    context_model:
      kmer_length: 3
      rules:
        - context: "GGG"
          sub_multiplier: 3.0
          indel_multiplier: 5.0
        - context: "GGC"
          sub_multiplier: 2.5
      profile_path: "context_errors.json"  # Alternative: load from file
```

## Success Criteria

- [x] `KmerErrorModel` struct in `src/core/seq_errors.rs` with `Vec<f32>`
      tables of size `4^k` for substitution and indel multipliers.
- [x] 2-bit per-base encoding: A=0, C=1, G=2, T=3.
- [x] Rolling hash: O(1) per position via left-shift, OR, and mask.
- [x] Inline YAML rules populate the table; all other k-mers default to 1.0.
- [x] JSON profile loading for learned profiles (T156 will produce these).
- [x] Applied by multiplying the per-base error probability before Bernoulli draw.
- [x] Unit test: configure GGG with 10× multiplier — error rate at GGG contexts
      is 8–12× the baseline.
- [x] Unit test: rolling hash produces same result as naïve k-mer lookup.
- [x] `cargo test` passes.
