# T118: Remove ContaminationConfig from v1 schema

**Priority**: P1
**Depends on**: —
**Status**: done

## Goal

`ContaminationConfig` is parsed from the YAML schema but never implemented.
Keeping it in the schema misleads users into thinking contamination simulation
is supported. Remove it from the v1 schema, or replace with a clear
"not yet implemented" error if the field is present.

## Steps

1. Locate `ContaminationConfig` in the schema and all references to it.
2. Remove the field from the v1 config struct, or add an error on parse if
   the field is set.
3. Update any example YAML files that reference contamination config.
4. `cargo fmt -- --check && cargo clippy -- -D warnings && cargo test`

## Success criteria

- [ ] `ContaminationConfig` is absent from the v1 schema, or triggers a clear error.
- [ ] No example YAML files reference the removed field.
- [ ] `cargo test` passes cleanly.
