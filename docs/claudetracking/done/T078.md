# T078: Allow user override of cfDNA fragment peak SDs; add source citation

**Epic**: EPIC-CFDNA-SEMANTICS
**Priority**: P2
**Depends on**: T077
**Status**: done

## Goal

The mono-nucleosomal SD of 20 bp and di-nucleosomal SD of 30 bp in `src/core/fragment.rs`
are hardcoded with no source. Add optional config overrides and a citation comment pointing
to a published cfDNA fragment distribution reference.

## Steps

1. Add `mono_sd: Option<f64>` and `di_sd: Option<f64>` to the cfDNA config block
2. In `CfdnaFragmentSampler::new`, use provided values or fall back to the defaults (20, 30)
3. Add a comment citing the Cristiano et al. 2019 Science paper (or Snyder et al. 2016) as
   the source for the default values
4. `cargo test`
