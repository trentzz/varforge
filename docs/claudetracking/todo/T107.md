# T107: Add strand concordance metric to sim_report.json

**Epic**: EPIC-TWIST-DUPLEX
**Priority**: P1
**Depends on**: T106
**Status**: todo

## Goal

Add a `strand_concordance` section to `sim_report.json` reporting, per variant, the fraction
of alt-carrying molecules with support in both AB and BA strands. The aggregate (across all
variants) should also be reported as a single number.

## Steps

1. In `WriterStats` (or the sim_report builder in `src/cli/simulate.rs`), compute per-variant:
   - `n_alt_mol`: total molecules carrying the alt (AB or BA)
   - `n_duplex_alt`: molecules with alt in both strands (from T106)
   - `strand_concordance`: `n_duplex_alt as f64 / n_alt_mol as f64`
2. Add these to the `variants` array in `sim_report.json`
3. Add an aggregate `overall_strand_concordance` field (weighted mean across all variants)
4. Add a unit test: check the JSON output contains `strand_concordance` for each variant
5. `cargo test`

## Success target

Overall strand concordance for spiked-in variants should be ≥ 0.99 (99%). Values below 0.9
indicate a bug in the variant spiking logic, not realistic noise.
