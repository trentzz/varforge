# T082: Include SV records in truth VCF

**Epic**: EPIC-TRUTH-VCF
**Priority**: P1
**Depends on**: none
**Status**: todo

## Goal

Wire `sv_vcf_info()` and `sv_vcf_alt()` from `src/variants/structural.rs:358-400` into the
truth VCF writer so that simulated SVs appear in the truth set. Currently SVs are applied to
reads but never written to the truth VCF.

## Steps

1. Read `src/variants/structural.rs` sv_vcf_info and sv_vcf_alt to understand the ALT and
   INFO fields produced
2. In `src/io/truth_vcf.rs`, add a `write_sv` method (or extend `write_variant`) for SV
   records
3. In `src/cli/simulate.rs`, after running the simulation, collect the applied SVs and pass
   them to the truth VCF writer
4. SVs should use the SVTYPE/SVLEN/END INFO format; ALT should use symbolic alleles
   (<DEL>, <DUP>, <BND>)
5. Add a unit test: a deletion SV produces a VCF record with SVTYPE=DEL and correct END
6. `cargo test`
