# T111: Add Twist capture uniformity parameters to preset and validate against targets

**Epic**: EPIC-TWIST-DUPLEX
**Priority**: P2
**Depends on**: T105
**Status**: todo

## Goal

Expose `coverage_cv_target` and `on_target_fraction_target` as optional fields in
`CaptureConfig` and report the achieved values in `sim_report.json`. The `twist` preset sets
targets of CV ≤ 0.25 and on-target ≥ 0.95. If the simulation falls short, emit a warning.

## Steps

1. Add `coverage_cv_target: Option<f64>` and `on_target_fraction_target: Option<f64>` to
   `CaptureConfig` in `src/io/config.rs`
2. In `sim_report.json`, add `achieved_coverage_cv` and `achieved_on_target_fraction`
   (compute from per-target coverage stats already tracked in WriterStats)
3. After the simulation, compare achieved values against targets and emit `tracing::warn!`
   if either target is missed
4. Wire `coverage_uniformity: 0.15` and `off_target_fraction: 0.03` into the `twist` preset
   (T105)
5. Add a unit test: at uniformity=0.0, the achieved CV is 0.0; at uniformity=0.5, CV > 0.1
6. `cargo test`
