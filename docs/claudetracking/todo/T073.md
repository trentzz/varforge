# T073: Reverse-complement R2 sequence and quality in BAM output

**Epic**: EPIC-BAM-FIDELITY
**Priority**: P0
**Depends on**: T056 (seq_utils module with reverse_complement)
**Status**: todo

## Goal

`src/io/bam.rs` sets the REVERSE flag on R2 but does not actually reverse-complement the
sequence or reverse the quality array before storing the record. Any tool that reads the BAM
will get wrong bases on R2 reads.

## Steps

1. In `write_pair` (and UMI variant), before setting the R2 record sequence, call
   `reverse_complement(&pair.seq2)` and store the result
2. Reverse `pair.qual2` before storing
3. Add a unit test: write a pair with known seq2, read back the BAM record, confirm the
   stored sequence is the reverse complement
4. `cargo test`

## Notes

The BAM spec requires that R2 is stored as the reverse complement of the biological sequence
when FLAG=REVERSE is set. If the caller does not do this, samtools will report the wrong
bases and tools expecting the biological sequence (via noodles or pysam) will get garbage.
