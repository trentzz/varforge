# T069: Replace intersect_with_targets O(n*m) with interval-based approach

**Epic**: EPIC-COVERAGE
**Priority**: P2
**Depends on**: none
**Status**: todo

## Goal

Replace the nested-loop O(n×m) implementation of `intersect_with_targets` in
`src/core/coverage.rs` with a sort-merge or interval-tree approach. Target: O((n+m) log n).

## Steps

1. Read the current implementation and identify all call sites
2. Sort both input slices by (chrom, start) if not already sorted
3. Implement a two-pointer merge: advance through genomic regions and targets in parallel,
   emitting intersections when they overlap
4. Confirm the output is identical to the current implementation on the existing tests
5. Add a benchmark (or at minimum a timing comment) showing the improvement for n=1000
   regions and m=5000 targets
6. `cargo test`

## Notes

This is a bottleneck for WES with 50,000 targets and large reference partitions. The naive
approach does 50,000 × 10,000 = 500M comparisons per sample.
