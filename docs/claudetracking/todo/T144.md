# T144: Add CI job for cargo audit

**Epic**: EPIC-POLISH (overallplans/EPIC-POLISH.md)
**Priority**: medium
**Depends on**: T133
**Status**: todo

## Goal

The CI pipeline (`ci.yml`) runs fmt, clippy, check, and test but does not run
`cargo audit`. Add an audit step so dependency security issues are caught
automatically.

## Success Criteria

- [ ] `.github/workflows/ci.yml` includes a `cargo audit` job.
- [ ] The job fails if `cargo audit` reports any errors (warnings are acceptable).
- [ ] The job runs on push to main and on PRs.
- [ ] `/update` has been run after changes.

## Steps

1. Add a new job to `ci.yml` that installs `cargo-audit` and runs it.
2. Use `cargo audit --deny warnings` or `cargo audit` with exit code check.

## Notes

After T133 lands, `cargo audit` should be clean. This job prevents regressions.
