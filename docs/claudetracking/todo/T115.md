# T115: Fix Normal::new().unwrap() panic when sd=0 in fragment.rs and quality.rs

**Priority**: P0
**Depends on**: —
**Status**: todo

## Goal

`Normal::new().unwrap()` panics at runtime when the standard deviation is zero.
Config validation allows `sd=0`, which means the panic is reachable from valid
user input. Fix by either rejecting `sd=0` in config validation or handling
zero-sd gracefully (returning the mean value directly).

## Steps

1. Locate all `Normal::new().unwrap()` calls in `fragment.rs` and `quality.rs`.
2. Add validation in config parsing to reject `sd=0` with a clear error message,
   or replace the unwrap with a fallback that returns the mean when `sd=0`.
3. Add a test covering the `sd=0` case to confirm no panic.
4. `cargo fmt -- --check && cargo clippy -- -D warnings && cargo test`

## Success criteria

- [ ] No reachable `unwrap()` panic when `sd=0` in fragment or quality config.
- [ ] Config validation emits a clear error, or the code handles `sd=0` gracefully.
- [ ] A test covers the `sd=0` path.
- [ ] `cargo test` passes cleanly.
