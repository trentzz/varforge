# T070: Wrap Config in Arc to eliminate per-region clones

**Epic**: EPIC-COVERAGE
**Priority**: P2
**Depends on**: none
**Status**: todo

## Goal

Wrap the `Config` passed into the rayon parallel region loop in an `Arc<Config>` so each
worker thread shares the config rather than cloning the full struct (which includes all
variant, clone, and copy-number Vecs).

## Steps

1. Identify the parallel region iteration in `src/cli/simulate.rs` (the rayon map over
   regions)
2. Wrap config in `Arc::new(cfg)` before the parallel section
3. Each `SimulationEngine::new` receives `Arc<Config>` instead of `Config`
4. Update `SimulationEngine` field type and constructor
5. Confirm no test failures: `cargo test`
6. `cargo clippy -- -D warnings`

## Notes

For large configs (many clones, many random mutations), each region clone allocates several
Vec buffers on the heap. With 100 regions and 10,000 random mutations this is 100 × 10,000
× sizeof(Variant) allocations before any reads are generated.
