## Goal

The vision docs require demonstrating that fgbio correctly groups UMI families
from VarForge duplex output. Generate a UMI-tagged BAM, run fgbio GroupReadsByUmi,
and verify that family sizes match expectations.

## Success Criteria

- [ ] VarForge generates a duplex UMI BAM for a small region (e.g. 100 kbp, 200x).
- [ ] fgbio GroupReadsByUmi runs successfully on the output.
- [ ] Family size distribution from fgbio is consistent with the configured family_size_mean.
- [ ] Results are saved to `benchmarking/downstreamvc/results/`.
- [ ] `/update` has been run after changes.

## Steps

1. Generate test data using the twist-umi-duplex preset.
2. Run fgbio GroupReadsByUmi.
3. Parse the family size histogram and compare with expectations.
