# T143: Avoid per-fragment reference clone in engine

**Epic**: EPIC-POLISH (overallplans/EPIC-POLISH.md)
**Priority**: medium
**Depends on**: none
**Status**: todo

## Goal

In `generate_region` (engine.rs line ~483), `pre_variant_frag_seq` clones the
entire fragment sequence on every iteration before variant spike-in. This is
needed for MD tag construction. Investigate whether the clone can be avoided
by computing MD tags from the cached region sequence directly, using the
fragment offset and CIGAR.

## Success Criteria

- [ ] The per-fragment clone of `pre_variant_frag_seq` is eliminated or reduced to only variant-carrying reads.
- [ ] MD tags in output BAM are identical to the current implementation (verified by comparing BAM output with a fixed seed).
- [ ] `cargo test` passes.
- [ ] `/update` has been run after changes.

## Steps

1. Profile the allocation cost of the clone at high coverage (e.g. 200x on 1 MB).
2. Compute ref_seq_r1 and ref_seq_r2 directly from region_seq using the fragment offset.
3. Only clone when a variant is actually applied to the fragment.
4. Verify MD tag correctness.

## Notes

At 200x on a 1 MB region with 150 bp reads, this clone allocates ~100 MB of
transient memory. Eliminating it would reduce GC pressure and improve throughput.
