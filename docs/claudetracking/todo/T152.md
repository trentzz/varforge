# T152: CycleErrorCurve — per-cycle error rate

**Epic**: EPIC-ERROR-MODEL
**Priority**: P1
**Depends on**: T151 (shared seq_errors module)
**Status**: todo

## Goal

Add a cycle-position-aware error rate that is independent of quality scores.
Models the gap between reported quality and actual instrument error rate, and
the characteristic 3' error rise in Illumina reads.

## Config API

```yaml
quality:
  sequencing_errors:
    base_error_rate: 0.001            # Flat per-base rate applied to all cycles
    cycle_error_model: "exponential"  # "flat", "exponential", or "custom"
    cycle_error_params:
      tail_start_fraction: 0.7
      tail_rate_multiplier: 5.0
    # For "custom": path to a two-column TSV (cycle, error_rate)
    cycle_error_tsv: "my_profile.tsv"
```

## Success Criteria

- [ ] `CycleErrorCurve` struct added to `src/core/seq_errors.rs`.
- [ ] `"flat"` mode: uniform rate across all cycles.
- [ ] `"exponential"` mode: rate rises exponentially from `tail_start_fraction`
      to `tail_rate_multiplier × base_error_rate` at the last cycle.
- [ ] `"custom"` mode: loads TSV, interpolates linearly between points.
- [ ] Curve is precomputed as `Vec<f64>` at model construction, not per read.
- [ ] Applied as an independent Bernoulli pass after quality-driven substitution.
- [ ] Unit test: flat mode error rate within ±10% of config value.
- [ ] Unit test: exponential mode — error rate at last cycle ≥ 4× first cycle.
- [ ] `cargo test` passes.
