# T075: Wire configurable MAPQ (not hardcoded 60)

**Epic**: EPIC-BAM-FIDELITY
**Priority**: P2
**Depends on**: none
**Status**: todo

## Goal

Replace the hardcoded MAPQ=60 in `src/io/bam.rs:139,224` with a configurable value. The
default should remain 60 for short reads, but long-read and error-prone reads benefit from
a different default or a per-read model.

## Steps

1. Add `mapq: u8` field to `OutputConfig` in `src/io/config.rs` (default 60)
2. Pass it through to the BAM writer
3. Optionally: for long-read mode, apply a simple read-quality-based MAPQ calculation
   (mean quality > 20 → MAPQ 60, else proportional)
4. Add a test: config with mapq=30 produces records with MAPQ=30
5. `cargo test`
