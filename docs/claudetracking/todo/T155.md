# T155: ErrorOrchestrator, SequencingErrorConfig, and preset defaults

**Epic**: EPIC-ERROR-MODEL
**Priority**: P1
**Depends on**: T151, T152, T153, T154
**Status**: todo

## Goal

Compose all error models into a single `ErrorOrchestrator` struct and wire it
into `build_read_pair` in `engine.rs`. Add the `SequencingErrorConfig` YAML
struct. Update platform presets with sensible error defaults.

## Success Criteria

- [ ] `ErrorOrchestrator` struct in `src/core/error_orchestrator.rs` (or
      `seq_errors.rs`) composes `CycleErrorCurve`, `KmerErrorModel`,
      `IndelErrorModel`, `StrandBiasModel`, `CorrelatedErrorModel`.
- [ ] `inject_all_errors(r1_seq, r1_qual, r2_seq, r2_qual, rng)` applies all
      passes in defined order.
- [ ] Orchestrator is constructed once per region, not per read pair.
- [ ] `SequencingErrorConfig` in `config.rs` under `QualityConfig` as
      `Option<SequencingErrorConfig>`. All fields optional with sane defaults.
- [ ] Validation rejects rates outside [0, 1], k outside [1, 5].
- [ ] When `sequencing_errors` is absent, behaviour is bit-for-bit identical
      to v0.1.x (same RNG calls, same output).
- [ ] Platform presets updated:
      `illumina_novaseq`: base_error_rate 0.001, indel_rate 0.00005,
      exponential cycle curve, GGG/GGC context rules.
      `pacbio_hifi`: indel_rate 0.001, flat curve.
      `nanopore_r10`: indel_rate 0.03, insertion_fraction 0.55.
- [ ] Existing integration tests pass unchanged.
- [ ] New integration test: NovaSeq preset produces substitution rate 0.08–0.15%
      and indel rate 0.003–0.008%.
- [ ] `cargo test` passes.
