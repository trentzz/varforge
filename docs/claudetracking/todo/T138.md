# T138: Audit and reduce dead_code annotations

**Epic**: EPIC-POLISH (overallplans/EPIC-POLISH.md)
**Priority**: medium
**Depends on**: none
**Status**: todo

## Goal

47 `#[allow(dead_code)]` annotations remain across the codebase. Audit each
one and either: (a) remove the annotation and make the code public/used,
(b) delete the dead code entirely, or (c) add a justification comment for
retaining it. The target is fewer than 15 annotations remaining, each with
a comment explaining why.

## Success Criteria

- [ ] Fewer than 15 `#[allow(dead_code)]` annotations remain.
- [ ] Every remaining annotation has a comment on the preceding line explaining why it is kept.
- [ ] `cargo clippy -- -D warnings` passes.
- [ ] `cargo test` passes.
- [ ] `/update` has been run after changes.

## Steps

1. Run `grep -rn "allow(dead_code)" src/` and categorise each into remove/delete/justify.
2. For `signatures.rs` module-level `#![allow(dead_code)]`, evaluate which items are actually used and remove the blanket allow.
3. Remove code that has no plausible future use.
4. Make code public that is genuinely useful but only called from tests.

## Notes

The vision docs targeted reducing from 69 to a justified minimum. Previous work
brought it from 69 to 47. This task continues that effort.
