# T145: Add doc tests for public API functions

**Epic**: EPIC-POLISH (overallplans/EPIC-POLISH.md)
**Priority**: low
**Depends on**: none
**Status**: todo

## Goal

`cargo test` reports "0 doc-tests". The library crate (`lib.rs`) exposes all
modules as public but has no documented examples. Add doc tests to at least
the most important public functions to serve as both documentation and
regression tests.

## Success Criteria

- [ ] At least 5 doc tests exist across the public API.
- [ ] Doc tests cover: `Config` loading, `SimulationEngine` construction, `Region` creation, `sample_alt_count`, and `NormalFragmentSampler`.
- [ ] `cargo test` reports at least 5 doc-tests passing.
- [ ] `/update` has been run after changes.

## Steps

1. Add `/// # Examples` sections to key public functions.
2. Each example should compile and run.
3. Run `cargo test --doc`.

## Notes

Doc tests serve double duty as both documentation and tests. They are the
most maintainable form of API documentation.
