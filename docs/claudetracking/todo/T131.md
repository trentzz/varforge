# T131: Add automatic BAM index (.bai) generation after BAM output

**Priority**: P2
**Depends on**: —
**Status**: todo

## Goal

VarForge does not generate a `.bai` index alongside the BAM output. Most
downstream tools (e.g. samtools, IGV, GATK) require an index for random
access. After writing the BAM, automatically generate the `.bai` index.

## Steps

1. After the BAM writer finishes, invoke the noodles (or equivalent) BAM
   indexing API to produce a `.bai` file adjacent to the BAM.
2. Log or report the index path.
3. Add a test that confirms a `.bai` file is created after simulation.
4. `cargo fmt -- --check && cargo clippy -- -D warnings && cargo test`

## Success criteria

- [ ] A `.bai` file is created automatically for every BAM output.
- [ ] A test asserts the `.bai` file exists after simulation.
- [ ] `cargo test` passes cleanly.
