# T141: Wire VariantTag into FASTQ output or remove

**Epic**: EPIC-POLISH (overallplans/EPIC-POLISH.md)
**Priority**: low
**Depends on**: none
**Status**: todo

## Goal

The `VariantTag` struct is populated for every read pair that carries a variant,
but no output path consumes it. Either:
(a) annotate FASTQ read names with variant info (e.g. `@read:VAR:chr1:1000:SNV`), or
(b) remove the population code and the struct.

Option (a) is more useful for debugging and downstream analysis.

## Success Criteria

- [ ] Either: FASTQ read names include variant annotation for reads carrying variants, OR the VariantTag population code is removed.
- [ ] If option (a): a test verifies that variant-carrying reads have the annotation in their name.
- [ ] If option (b): `VariantTag` struct and all population code is removed; `#[allow(dead_code)]` on the struct is gone.
- [ ] `cargo test` passes.
- [ ] `/update` has been run after changes.

## Steps

1. Decide between option (a) and (b).
2. Implement.
3. Test.
