# T053: Fix quality averaging u8 overflow in spike_in.rs

**Epic**: EPIC-CORRECTNESS
**Priority**: P0
**Depends on**: none
**Status**: todo

## Goal

Fix plain u8 addition overflow in `src/variants/spike_in.rs:94`. In debug builds this panics;
in release it wraps silently. Add a test covering values that would overflow.

## Steps

1. Find all plain `u8 + u8` quality averaging expressions in `spike_in.rs`
2. Replace with widening: `((a as u16 + b as u16) / 2) as u8`
3. Add a unit test that passes qualities [200, 200] and checks the average is 200, not 0 or
   panicking

## Notes

Normal Phred scores (0–41) don't overflow, but empirical profiles from high-quality PacBio
reads stored as raw u8 can exceed 127.
