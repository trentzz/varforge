# T130: Add module-level documentation to undocumented modules

**Priority**: P2
**Depends on**: —
**Status**: todo

## Goal

22 modules in `src/` lack module-level documentation comments. Add a `//!`
doc comment to each undocumented module describing its purpose, main types,
and how it fits into the simulation pipeline.

## Steps

1. Identify all modules missing `//!` doc comments (`cargo doc --no-deps 2>&1`
   or by inspection).
2. Add a concise `//!` block to each module covering:
   - What the module does.
   - Key public types or functions.
   - Any important constraints or design notes.
3. `cargo doc --no-deps` should produce no missing-doc warnings for modules.
4. `cargo fmt -- --check && cargo clippy -- -D warnings && cargo test`

## Success criteria

- [ ] All modules in `src/` have a `//!` doc comment.
- [ ] `cargo doc --no-deps` completes without module-level doc warnings.
- [ ] `cargo test` passes cleanly.
