# T113: Fix UMI RX:Z tag not written to BAM in production path

**Priority**: P0
**Depends on**: —
**Status**: todo

## Goal

`write_pair_with_umi` is never called from the main simulate loop. As a result,
UMI sequences are never written to the `RX:Z` tag in BAM output, even when UMI
mode is configured. Fix the simulate loop to call `write_pair_with_umi` when a
UMI is present.

## Steps

1. Locate where read pairs are written in the simulate loop.
2. Confirm `write_pair_with_umi` exists and is correct.
3. Thread UMI values through to the write call and invoke it when UMI mode is active.
4. Add a test that runs with UMI config and asserts `RX:Z` is present in output BAM records.
5. `cargo fmt -- --check && cargo clippy -- -D warnings && cargo test`

## Success criteria

- [ ] `write_pair_with_umi` is called from the simulate loop when UMI mode is enabled.
- [ ] A test confirms `RX:Z` is set in BAM output when UMI mode is on.
- [ ] `cargo test` passes cleanly.
