# T151: IndelErrorModel — sequencing indel injection

**Epic**: EPIC-ERROR-MODEL
**Priority**: P1
**Depends on**: none
**Status**: todo

## Goal

Implement per-base sequencing indel injection. Real instruments produce
insertions and deletions alongside substitutions at platform-specific rates.
Without indels, VarForge reads are trivially distinguishable from real data.

## Config API

```yaml
quality:
  sequencing_errors:
    indel_rate: 0.00005       # Per-base probability of an indel error
    indel_insertion_fraction: 0.5  # Fraction that are insertions (rest deletions)
    max_indel_length: 3       # Max length; lengths drawn from Geometric(0.7)
```

## Success Criteria

- [ ] New file `src/core/seq_errors.rs` with `IndelErrorModel` struct and
      `inject_indel_errors(seq, qual, model, rng)` function.
- [ ] Indels are injected in reverse-position order to avoid index shift bugs.
- [ ] After injection, seq and qual are truncated or padded to `read_length`
      (fixed-length Illumina contract). Insertions trim the 3' end; deletions
      pad with N/Q0.
- [ ] Unit test: 10000 reads at `indel_rate: 0.1` — observed rate within
      [0.09, 0.11].
- [ ] Unit test: length distribution follows Geometric(0.7), capped at
      `max_indel_length`.
- [ ] Unit test: insertion fraction respected within ±5%.
- [ ] `cargo test` passes.
