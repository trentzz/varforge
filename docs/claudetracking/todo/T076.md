# T076: Fix inverted ctDNA fraction in cfDNA fragment sampler

**Epic**: EPIC-CFDNA-SEMANTICS
**Priority**: P0
**Depends on**: none
**Status**: todo

## Goal

Fix `src/core/engine.rs:257` where `ctdna_fraction = 1.0 - purity`. In cfDNA, higher tumour
burden (purity) means more ctDNA, so the formula is inverted. After the fix, a simulation
with purity=0.05 (5% ctDNA) should produce mostly normal-cell fragment lengths.

## Steps

1. Read the cfDNA fragment model in `src/core/fragment.rs` to understand how ctdna_fraction
   is used (likely controls the mixing weight between mono- and di-nucleosomal peaks)
2. Fix the formula to `ctdna_fraction = purity` (or use the new T077 explicit field if
   available)
3. Add a regression test: at purity=0.01 (1% ctDNA), the fraction of fragments in the
   mononucleosomal peak should be approximately 0.01, not 0.99
4. `cargo test`

## Notes

This is a sign-flip bug that inverts all cfDNA simulations. It would cause ctDNA callers
benchmarked against VarForge output to appear to perform opposite of reality.
