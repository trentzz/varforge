# T072: Add NM (edit distance) tag to generated BAM records

**Epic**: EPIC-BAM-FIDELITY
**Priority**: P1
**Depends on**: none
**Status**: todo

## Goal

Set the NM tag on every generated BAM record. NM is required by Mutect2 and VarDict and
flagged as an error by Picard ValidateSamFile.

## Steps

1. In `write_pair` (and `write_pair_with_umi`, `write_single_read`), count the number of
   mutations applied to each read from the `ReadPair.variant_tags` (or from a per-read
   mismatch count passed in)
2. Set `NM:i:<count>` in the record's data fields using noodles sam data
3. For reads with no variants: NM=0
4. Add a unit test: a read with one SNV has NM=1; a read with no variants has NM=0
5. `cargo test`

## Notes

NM counts mismatches + inserted + deleted bases relative to the reference. For simple SNVs
it equals the number of SNVs. For indels it equals the net length difference.
