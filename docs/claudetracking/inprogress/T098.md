# T098: Parallelise multi-sample runs across samples

**Epic**: EPIC-MULTI-SAMPLE
**Priority**: P2
**Depends on**: none
**Status**: todo

## Goal

`src/cli/simulate.rs` runs each sample in a multi-sample config sequentially. Samples are
independent and can run in parallel, cutting wall time proportionally to the number of samples.

## Steps

1. Identify the sequential sample loop in `run_multi_sample` (or equivalent)
2. Wrap it in a `rayon::iter::IntoParallelIterator` with each sample owning its own
   output directory and rng seed
3. Ensure per-sample progress logging still works (use `tracing::info` inside the parallel
   block — it is thread-safe)
4. Add a test: 3-sample config completes and all three output directories contain BAM files
5. `cargo test`

## Notes

Each sample needs its own `SimulationEngine` instance (not shared). The rayon thread count
is already controlled by the `--threads` flag.
