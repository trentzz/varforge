# T156: ProfileLearner extension — indel and cycle error rates

**Epic**: EPIC-ERROR-MODEL
**Priority**: P2
**Depends on**: T151, T152, T153, T155
**Status**: in_progress

## Goal

Extend `varforge learn-profile` to extract sequencing indel rates and per-cycle
error rates from real BAMs. The output JSON gains new fields that, when loaded,
auto-configure the full `ErrorOrchestrator`.

## Success Criteria

- [ ] `LearnerStats` gains `insertion_counts_by_cycle`, `deletion_counts_by_cycle`,
      `bases_by_cycle` fields for CIGAR-based indel rate extraction.
- [ ] During BAM scan, CIGAR `I` and `D` operations increment the per-cycle
      indel counters (MAPQ ≥ 30 only; skip positions in known-variant VCF if
      provided).
- [ ] `ProfileJson` gains optional `indel_error_profile` and `cycle_error_rates`
      fields (all `#[serde(default)]` for backward compatibility).
- [ ] `EmpiricalQualityModel::from_profile` constructs an `ErrorOrchestrator`
      when these fields are present.
- [ ] Old profile JSONs (without new fields) load correctly with zero-rate
      defaults for the new features.
- [ ] Unit test: synthetic BAM with known indel rate — learner extracts rate
      within ±20%.
- [ ] `cargo test` passes.
