## Goal

Execute the downstream variant caller benchmark designed in
`benchmarking/downstreamvc/design.md`. Generate VarForge BAMs with known
variants, run Mutect2, and measure sensitivity and precision against the
truth VCF. This is the primary quality gate for release.

## Success Criteria

- [ ] VarForge generates a BAM and truth VCF for at least one benchmark config (e.g. chr22, 30x, 100 SNVs).
- [ ] Mutect2 runs successfully on the generated BAM.
- [ ] `evaluate.py` computes sensitivity and precision.
- [ ] Sensitivity >= 80% for SNVs with VAF >= 0.1 at 30x coverage.
- [ ] Results are saved to `benchmarking/downstreamvc/results/`.
- [ ] Any VarForge bugs discovered during validation are filed as separate tasks.
- [ ] `/update` has been run after changes.

## Steps

1. Review `benchmarking/downstreamvc/scripts/run_benchmark.sh` and adapt if needed.
2. Generate test data with VarForge using a chr22 config.
3. Run Mutect2 on the output BAM.
4. Run `evaluate.py` against the truth VCF and Mutect2 calls.
5. Record results.

## Notes

Requires hg38 reference and Mutect2 installed. If these are not available
locally, document what is needed and mark as blocked.
